<text>
Done. Here's a summary of the fix:
**Root cause:** In `InlineModelAdmin`, the `has_add_permission`, `has_change_permission`, and `has_delete_permission` methods for auto-created m2m through models all delegated to `self.has_view_permission()`. Since `has_view_permission` returns `True` if the user has **either** view or change permission on the related model, a view-only user was treated as having full edit access to the m2m relationship.
**Fix:** Changed all three edit-related permission checks to directly verify the **change** permission on the related model (e.g., `Photo`) rather than delegating through `has_view_permission`. Extracted the "find the related model's opts" logic into a shared `_get_auto_created_related_opts()` helper to reduce duplication.
**Result:**
- View-only users on the related model → can **view** the inline but not add/change/delete
- Change users on the related model → can view AND edit the inline (unchanged)
- `has_view_permission` still accepts both view and change perms (unchanged)
</text>
