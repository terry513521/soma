The fix works. The issue was that `middle[len(stripped):]` used the character position from the unescaped string to index into the original string, which fails when HTML entities like `&lt` expand to fewer characters. The fix instead calculates how many trailing characters were removed from the unescaped string and removes that same count from the end of `middle`.
